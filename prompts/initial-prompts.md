Preferable building the same toy application using different architectures and showcasing ALL different patterns involved a senior software engineer should know about.

What I want you to do is first create the requirements for such an application. It should be the simplest possible and so complex at the same time that will present all the patterns involving those architectures, their similarities and their differences.
If the app is too simple, it would not showcase the real benefits and cons of each design.

Your design requirement would be places inside [docs](docs/) folder in a prd.md file that would be used later to create the implementation for the application.

# Folder Conventions

Use one top-level folder per architecture solution.

## Expected Shape

```text
<architecture-folder>/
  lessons/
    001-...
    002-...
  ...
```

## Naming Guidance

Prefer stable, readable folder names derived from the architecture name.

Examples:

- `layered-architecture`
- `hexagonal-architecture`
- `clean-architecture`
- `onion-architecture`

## Lessons Folder

Each architecture folder should contain:

- `lessons/`

Lesson files should be incrementally numbered and kept in order.

# Orchestrator Output

Use this structure when explaining the current pipeline state.

## Current State

- what artifacts exist
- what is strongest
- what is missing
- what is weak or contradictory

## Recommended Next Step

- which skill should run next
- why it is the highest-leverage next action

## Optional Alternate Path

If there is a reasonable alternate route, mention it briefly.

## Completion State

When complete, summarize:

- final artifact set
- residual risks
- readiness for implementation

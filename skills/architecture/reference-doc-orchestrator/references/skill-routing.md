# Skill Routing

Use this map to decide which specialist skill to invoke.

## `$requirements-discovery-interviewer`

Use when:

- the idea is vague
- workflows and actors are not yet clear

## `$domain-glossary-extractor`

Use when:

- terminology is unstable
- synonyms and overloaded words are causing drift

## `$requirements-gap-analyzer`

Use when:

- artifacts exist but are incomplete or contradictory
- you need the next best clarifying questions

## `$prd-author`

Use when:

- discovery is strong enough to write a product requirements document

## `$canonical-domain-modeler`

Use when:

- PRD and vocabulary are stable enough to derive core domain semantics

## `$canonical-use-case-modeler`

Use when:

- domain semantics are stable enough to derive application-layer behavior

## `$canonical-contract-author`

Use when:

- the use-case model is stable enough to map to external behavior

## `$acceptance-scenario-author`

Use when:

- the required behaviors are clear enough to express as reusable scenarios

## `$architecture-readiness-reviewer`

Use when:

- the artifact set appears complete
- or confidence is low and a gate review is needed before implementation

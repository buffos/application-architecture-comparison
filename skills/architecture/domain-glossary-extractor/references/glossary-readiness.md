# Glossary Readiness

The source material is ready for glossary extraction when most of the following are true:

- the core workflow can be described
- the main actors are known
- at least some business objects recur across the material
- important terms are visible even if not fully normalized yet

Not ready signals:

- the conversation is still too abstract
- the user cannot name the main business objects
- terminology changes wildly from sentence to sentence
- the material is only UI or architecture talk with little business language

If not ready, ask for examples and concrete workflow descriptions before normalizing terms.

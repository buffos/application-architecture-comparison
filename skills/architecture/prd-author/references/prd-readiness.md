# PRD Readiness

The source material is ready for PRD drafting when most of the following are true:

- the core problem is explicit
- the main actors are known
- at least one end-to-end workflow is understood
- scope is bounded
- non-goals are visible
- important rules and constraints are partially concrete
- at least one failure or exception path is known
- success criteria are not purely technical

Not ready signals:

- the user only described screens
- the business problem is still ambiguous
- the workflow depends on undefined terms
- all rules are described as “validation”
- the product boundary is unknown
- the user keeps changing the primary goal

If not ready, ask focused follow-up questions instead of drafting.

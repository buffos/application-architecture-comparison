# Scenario Coverage

Aim to cover these dimensions when the product supports them.

## Core Flow Coverage

- normal happy path
- standard business completion

## Rule Coverage

- threshold-triggered review
- category-based restriction
- role-based approval

## Failure Coverage

- invalid input or state
- insufficient stock or blocked resource
- policy rejection

## Reversal Coverage

- cancellation
- return
- refund

## Visibility Coverage

- approval queue visibility
- low stock reporting
- conversion or return metrics

## Extensibility Coverage

- plugin-enabled pricing or policy variation
- configurable rule outcome changes

# Use-Case Readiness

The inputs are ready for canonical use-case modeling when most of the following are true:

- the main business workflows are known
- important actors and intents are known
- the core domain concepts are named
- the main state transitions are visible
- business rules and exception paths are explicit enough to affect orchestration

Not ready signals:

- the requirements are still mostly UI-driven
- commands cannot be named without guessing
- the domain model is still unstable
- no one can say which workflows are reads versus writes
- failure paths are still vague

If not ready, ask follow-up questions before drafting.
